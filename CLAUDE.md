# RelocationHub

SaaS app helping expats relocating to the Netherlands organize their move. Two services: Next.js frontend (Vercel) + FastAPI backend (Railway) + Supabase for DB/auth/storage.

## Architecture

```
frontend/   → Vercel (relocation-hub.vercel.app)
backend/    → Railway (relocation-hub-production.up.railway.app)
```

- Auth: Supabase Google OAuth
- AI: Anthropic Claude (checklist generation) via backend only
- Storage: Supabase bucket `documents` (RLS enforced)
- Cron: Vercel cron hits `/api/keepalive` every 5 min to prevent Railway cold starts

## Env vars — where they live

| Variable | Location |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Vercel |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Vercel |
| `NEXT_PUBLIC_API_URL` | Vercel |
| `SUPABASE_URL` | Railway |
| `SUPABASE_ANON_KEY` | Railway |
| `SUPABASE_SERVICE_KEY` | Railway |
| `ANTHROPIC_API_KEY` | Railway |
| `FRONTEND_URL` | Railway |

Never put `NEXT_PUBLIC_*` in Railway. Never put `FRONTEND_URL` in Vercel.

## Key constraints

- All pages that touch Supabase need `export const dynamic = 'force-dynamic'` to prevent build-time prerender errors
- CORS in `backend/app/main.py` allows Vercel origin + localhost only
- SA users get hardcoded VFS prerequisite tasks first (priority 100), then document tasks (priority 90) — these are never generated by Claude
- Dashboard locks non-critical task sections until all critical tasks are ticked off (dependency lock)
- Claude prompt: employer applies to IND on user's behalf — do not say user applies; "critical" category reserved for hardcoded tasks only

## Task categories (fixed order)

`critical → visa → admin → employment → housing → banking → healthcare → transport → shipping → pets`

## Current state

Working end-to-end: Google OAuth → onboarding → AI checklist → dashboard.

Not yet built: Stripe, email reminders, calendar integration, documentation portal, API rate limits, HR contact notifications.

## Roadmap

### Phase 1 — Engagement layer (active)

1. **HR/consultant contact** — user adds a contact person during onboarding; contact gets email on task completions + weekly digest (Resend). Key differentiator.
2. **Reminders** — per-task due dates, user-configurable email cadence via Resend
3. **Calendar integration** — Google Calendar + iCal export for deadline tasks
4. **Documentation portal** — upload/manage docs per task (Supabase Storage already provisioned)
5. **Claude API rate limits** — per-user daily cap, usage counter in Supabase

### Phase 2 — Monetisation

- Stripe €3.99/mo
- Free: checklist only — Paid: docs + AI validation + reminders

### Phase 3 — B2B white-label

- Admin portal: HR/consultant sees all assigned users' progress, sets reminders, annotates tasks
- Sold to relocation agencies and corporate HR teams
