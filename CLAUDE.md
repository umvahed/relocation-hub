# RelocationHub

SaaS app helping expats relocating to the Netherlands organize their move. Two services: Next.js frontend (Vercel) + FastAPI backend (Railway) + Supabase for DB/auth/storage.

## Architecture

```
frontend/   → Vercel (relocation-hub.vercel.app)
backend/    → Railway (relocation-hub-production.up.railway.app)
```

- Auth: Supabase Google OAuth + email/password
- AI: Anthropic Claude (checklist generation + document validation + risk score) via backend only
- Storage: Supabase bucket `documents` (RLS enforced)
- Cron: Vercel cron hits `/api/keepalive` every 5 min to prevent Railway cold starts

## Key constraints

- All pages that touch Supabase need `export const dynamic = 'force-dynamic'` to prevent build-time prerender errors
- CORS in `backend/app/main.py` allows Vercel origin + localhost only
- SA users get hardcoded VFS prerequisite tasks first (priority 100), then document tasks (priority 90) — these are never generated by Claude
- Dashboard locks non-critical task sections until all critical tasks are ticked off (dependency lock)
- Claude prompt: employer applies to IND on user's behalf — do not say user applies; "critical" category reserved for hardcoded tasks only
- GDPR: document bytes are never stored in our DB or logged — only validation results (JSON). Consent stored in `profiles.ai_validation_consent` + `ai_validation_consent_at`
- Document validation supports PDF + images only (jpeg/png/gif/webp) — Word/DOCX returns 422
- `api_usage` table has a `call_type` column — checklist/validation/risk_score each have independent daily limits

## Task categories (fixed order)

`critical → visa → admin → employment → housing → banking → healthcare → transport → shipping → pets`

## Current state

Working end-to-end: Google OAuth → onboarding → AI checklist → dashboard → document upload + iCal feed.

Not yet built: Document AI Validation, Relocation Risk Score, Stripe, email reminders, HR contact notifications.

## Roadmap

### Phase 1 — Premium AI features (active)

1. **Document AI Validation** — upload passport/contract/degree, Claude validates against IND 2025 requirements. Gated behind paid tier. GDPR consent modal, no raw content stored. See `PLAN.md`.
2. **Relocation Risk Score** — 0-100 score across 4 dimensions (critical completion 40%, timeline 30%, docs 20%, profile 10%). Top 3-5 risk items. Python calculates scores; Claude writes risk item text. See `PLAN.md`.
3. **IND Processing Time Intelligence** — crowdsource/scrape real IND processing times; alert users if move deadline is at risk.

### Phase 2 — Monetisation

- Stripe €3.99/mo
- Free: checklist only — Paid: document validation + risk score + reminders
- Pre-Stripe: tier manually granted via `POST /api/admin/grant-paid-tier` (X-Admin-Secret header, ADMIN_SECRET env var)

### Phase 3 — Engagement layer

- Email reminders — per-task due dates, configurable cadence via Resend
- HR/consultant contact notifications — weekly digest + task completion alerts
- Calendar integration — Google Calendar + iCal export (iCal feed already built)

### Phase 4 — B2B white-label

- HR/Company Portal: companies pay per-employee, HR sees all relocatees' progress
- Admin portal: consultant sees assigned users, sets reminders, annotates tasks
- Sold to relocation agencies and corporate HR teams
